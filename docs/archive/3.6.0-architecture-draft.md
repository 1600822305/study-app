## 二、设计原则

### 2.1 3.6.0 的本质：**思维操作系统**

不是：
- ❌ 工具罗列（那是工具栏）
- ❌ 流程手册（那是步骤说明书）
- ❌ 知识点清单（那是字典）

而是：
- ✅ **思维框架**——遇到任何导数题，大脑应该如何运作
- ✅ **决策树**——在每个分支点告诉你"下一步看什么"
- ✅ **共享语言**——3.6.1-3.6.5 的所有页面都用相同术语和流程

## 三、3.6.0 内容架构（6 大模块）

### 模块 A: 🗺 导数大题的"题型识别地图"

**回答**："这道题本质是什么？我应该用哪种思路？"

所有高考导数大题，归为 **3 大类**：

| 类型 | 题目特征 | 标准切入流程 | 对应子页面 |
|---|---|---|---|
| **A 类：研究 `f` 的性质** | "求单调区间 / 极值 / 最值 / 值域" | 求 `f'` → 切点 → 判号 → 写性质 | 3.6.1 / 3.6.2 / 3.6.3 |
| **B 类：通过 `f` 求参数** | "对所有/存在 x，使..." 求 `a` 范围 | 构造 `g(x)` → 求 `g` 性质 → 反推 `a` | 3.6.4 / 3.6.5 |
| **C 类：用导数证不等式** | "证明 `f(x) ≥ g(x)`" | 构造差函数 `h(x)=f-g` → 求 `h` 最值 → 证 `h≥0` | （后续章节） |

**关键洞察**：
- **B、C 类的核心步骤都是"研究某函数的性质"**——全归到 A 类的方法
- **3.6.0 真正讲的是 A 类的标准流程**，B/C 类只需补"如何构造辅助函数"

### 模块 B: 🎯 "判 `f` 单调性" 的标准 4 步

**回答**："不管题目多复杂，第一步先做什么？"

**无论 A/B/C 哪类，前 4 步完全一致**：

```
① 确定定义域
   - 含 ln 真数 > 0
   - 含分母 ≠ 0
   - 含 √ 根号内 ≥ 0
   
② 求导并整理
   - 多项式 → 因式分解
   - 分式 → 通分成 分子/分母
   - 含 e^x 等恒正因子 → 直接擦掉（见模块 C）

③ 找切点（见模块 C 的切点扫描）
   - 驻点：解 f'(x) = 0
   - 断点：让 f 自己消失的点

④ 分段判 f' 正负
   - 切点把定义域切成若干段
   - 每段任取一值 / 看因子同号异号
```

**这 4 步是 3.6.1 的核心，也是 3.6.2-3.6.5 的"通用前置步骤"**。

### 模块 C: ✂️ 切点扫描法（解决痛点 P1）

**回答**："分割定义域的点从哪来？"

#### C.1 切点的本质定义

> **切点 = 让 `f'` 变号或让 `f` 不连续的所有位置**

#### C.2 切点的两大来源

| 来源 | 怎么找 | 例子 |
|---|---|---|
| **① 驻点（`f'` 的零点）** | 解 `f'(x) = 0` | `f' = 3x²-3` → 驻点 `x=±1` |
| **② 断点（`f` 不连续点）** | 看 `f(x)` 的"危险信号" | （见 C.3） |

#### C.3 断点信号识别表（必背）

| 信号 | 断点是 | 例子 |
|---|---|---|
| **分母** | 让分母 = 0 的点 | `1/x` → `x=0`；`1/(x-2)` → `x=2` |
| **`ln □`** | 让真数 = 0 的边界 | `ln x` → `x=0`；`ln(x-1)` → `x=1` |
| **`√□`** | 让根号内 = 0 的边界 | `√x` → `x=0`；`√(x-3)` → `x=3` |

#### C.4 不产生切点的"安全函数"

| 函数 | 为什么安全 |
|---|---|
| `e^x` | 永远 > 0，永远有定义 |
| 多项式 | 永远有定义 |
| `sin x`, `cos x` | 永远 -1 ≤ 值 ≤ 1，永远有定义 |

#### C.5 切点扫描 3 步法

```
① 看 f(x) 长什么样：有分母吗？有 ln 吗？有 √ 吗？
② 解 f'(x) = 0 找驻点
③ 把所有切点排在数轴上 → 把定义域切成若干段
```

#### C.6 关键澄清（学生最容易混的）

> **切点 ≠ 函数零点**
> - **切点**：让 `f'` 变号或 `f` 消失的点
> - **零点**：函数值 = 0 的点（函数还存在）
>
> **多项式（如 `x³-3x`）只有驻点，没有断点**——这是大多数题的常态

### 模块 D: 🌳 分类讨论决策树（核心，解决痛点 P2-P5）

**回答**："含参题怎么分类？分多少类？怎么知道不漏？"

#### D.1 何时需要分类（Q1）

```
求出 f'(x) 后，问自己：

   f' 里含参数吗？
   ├── 不含 → 不需分类，直接走标准 4 步
   └── 含   → 进入 D.2 决定按什么分
```

#### D.2 按什么分（Q2，4 层级口诀）

> **从外往内问 4 层，问到"不含参"那一层就停**

| 层级 | 触发条件 | 分法 | 类数 |
|---|---|---|---|
| **第 1 层：二次项系数含参** | `f' = ax² + ...` 中 `a` 含参 | `a > 0` / `= 0` / `< 0` | 通常 3 类 |
| **第 2 层：判别式含参** | `Δ = a²-4` 等 | `Δ > 0` / `= 0` / `< 0` | 通常 3 类 |
| **第 3 层：两根大小含参** | 两根 `1` 和 `a`，谁大？ | `1 < a` / `=` / `> a` | 通常 3 类 |
| **第 4 层：根与端点位置含参** | 根 `a` 与区间 `[0,1]` 关系 | 根 `< 0` / 在区间内 / `> 1` | 通常 3 类 |

**口诀**：**系数 → Δ → 两根 → 端点**

**实战例子**（用 3.6.1 现有例题对照）：

- 3.6.1 例 1（`f'=(ax-1)/x`）：第 1 层只问到"a 是否为 0"，**2 类**
- 3.6.1 例 2（`f'=6(x-1)(x-a)`）：跳过第 1、2 层，第 3 层比较 `1` 与 `a`，**3 类**
- 3.6.1 例 3（`f'` 同例 2，但区间 `[0,1]`）：又叠加第 4 层，**3 类**
- 3.6.1 例 4（`f'=(x²-ax+1)/x`）：第 2 层 `Δ=a²-4`，**3 类**（`Δ≤0` 合并 + `Δ>0` 时按根的位置）

#### D.3 分多少类（Q3）

每层"通常 3 类"是默认值。但有时层级之间会**叠加**（如例 3 = 第 3 层 + 第 4 层），导致总类数 > 3。

**简单原则**：层级数确定后，每层独立分类，分类数相乘 = 总类数。

#### D.4 不漏不重的 3 道自检关卡（Q4、Q5）

```
关卡 1：临界值单列了吗？
  - a=0、Δ=0、a=1（重根边界）等"过渡值"必须单独成一类
  - 不能并入两侧（"≤" 还是 "<" 一定有讲究）

关卡 2：所有 a 取值都覆盖了吗？
  - 数轴画一遍参数 a 的范围
  - 每段都对应一种分类，没有空隙

关卡 3：各类之间不重叠？
  - 边界点只能属于一类
  - 写 "a≤0" 和 "a>0"（边界 0 归左）
  - 不能写 "a≤0" 和 "a≥0"（边界 0 重复）
```

### 模块 E: 📝 答题规范（全章共享）

**回答**："写出来怎么才不丢分？"

#### E.1 标准答题模板

```
解：
① 定义域：______
② 求导：f'(x) = ______
③ 分类讨论：
   情况一：当 a ___ 时，______
   情况二：当 a ___ 时，______
   ...
④ 综上所述：
   - a ___ 时，f(x) 的单调区间为 ______
   - a ___ 时，f(x) 的单调区间为 ______
```

#### E.2 高频扣分点（10 条，迁自当前 3.6.1 末尾）

```
① 没写定义域
② 求导/化简错（连环扣）
③ 没声明 f' 符号由谁决定（分式题必先说"分母恒正/负，符号由分子定"）
④ 没声明分类标准（开头要写"按 a 分类"或"按 Δ 分类"）
⑤ 漏掉边界/临界值（a=0 等必须单独列）
⑥ 分类不互斥或不完备
⑦ 重根当变号根（Δ=0 时不变号）
⑧ 单调区间用 ∪ 连接（不能并集）
⑨ "增/减函数" 与 "增/减区间" 混淆
⑩ 缺综上 / 结论不规范
```

### 模块 F: 🆘 零点解不出的两大救命招（进阶）

**回答**："如果 `f'(x)=0` 解不出 `x`，怎么办？"

#### F.1 二阶导反推法

```
① 求 f''(x)
② 用 f'' 判 f' 的单调性
③ 找 f' 的最值
④ 由 f' 单调性 + 端点值反推 f' 的零点位置
⑤ 继续走标准 4 步的 ④
```

**例**：`f'(x) = e^x - 2x - 1` 解不出
- `f''(x) = e^x - 2 = 0` 得 `x = ln 2`
- `f'` 在 `(-∞, ln 2)` 减、`(ln 2, +∞)` 增
- `f'(ln 2) = 2 - 2ln 2 - 1 = 1 - 2ln 2 ≈ -0.39 < 0`
- `f'(0) = 0`，`f'(2) = e²-5 > 0`
- 由连续性：`f'` 在 `(0, ln 2)` 内变号一次 = 1 个零点；`(ln 2, 2)` 内变号一次 = 1 个零点

#### F.2 设而不求 + 韦达定理

```
① 设两根为 x₁, x₂（不解出来）
② 用韦达定理表达 x₁+x₂、x₁·x₂
③ 用根与系数的关系反推 x₁、x₂ 的正负、大小
④ 因式分解 f' = (x-x₁)(x-x₂) → 判号
```

**例**（迁自 3.6.1 例 4）：`f' = (x²-ax+1)/x`
- 设两根 x₁, x₂，韦达：x₁+x₂ = a，x₁·x₂ = 1
- `a > 2`：积 > 0 且 和 > 0 → 两正根
- `a < -2`：积 > 0 且 和 < 0 → 两负根
- 不需要求出具体根，照样能判号

## 七、设计自检（深度反思）

### 7.1 这个架构会不会"过度工程"？

**风险**：3.6.0 内容太多，学生看不下去；新增页面学习负担反而增大。

**应对**：
- 严格执行"必须 ≥ 2 个子页面用"的内容选取原则
- 模块 D 决策树是核心，其他模块可以做得**极简**（速查表风格）
- 整个 3.6.0 控制在 **30-40 分钟阅读量**

### 7.2 决策树真的能解决痛点吗？

**质疑**：学生的痛点是"做题时不知道按什么分"，给一棵决策树就够了？

**反思**：决策树只是"思维框架"。**真正的内化需要练习**。所以 3.6.0 的决策树必须配 4 个迷你例子，让学生**用决策树走 4 遍**，建立肌肉记忆。

**建议**：每个例子的格式严格固定为：

```
迷你例 X：f(x) = ___
① 求 f' = ___
② 含参吗？___
③ 第几层？___
④ 分几类？___
（不写完整解答，只走决策树）
```

### 7.3 如果 3.6.0 没人看怎么办？

**风险**：学生直接进 3.6.1，跳过 3.6.0。

**应对**：
- 在 3.6.1 顶部明显位置写"⚠️ 第一次学请先看 3.6.0"
- 在每个 3.6.x 子页面遇到工具时**主动指向 3.6.0**（"切点扫描详见 3.6.0 §C"）
- HomePage 的章节卡片在 3.6.1 前显示 3.6.0

### 7.4 内容会不会与已有的 3.4、3.5 重复？

**风险**：3.4 教过求导公式，3.5 教过切线方程，3.6.0 又讲一遍？

**核查**：
- 模块 B 的"求导技术" 只提**指导原则**（哪种方法用哪种），具体公式引用 3.4
- 不重新讲公式，只讲**应用决策**

## 八、核心断言

> **3.6.0 不是"工具集"，是"导数大题的思维操作系统"**
> **它的价值不在于"有多少东西"，而在于"让学生学会思考"**

如果做好了，**3.6.1-3.6.5 的总篇幅可以减少 30%**——因为大量重复的"工具讲解"被抽象到一处。

如果做不好，**3.6.0 会变成又一个累赘的页面**——学生宁愿背例题也不看抽象方法论。

成败关键：**模块 D 分类讨论决策树 + 4 个迷你例子能不能让学生"看完就会"**。
