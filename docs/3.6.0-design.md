## 四、设计决策记录

### 4.1 为什么不叫"通用方法论"？

**初版**曾设想 3.6.0 包含：三步走思维 / 切点扫描 / 分类讨论决策树 / 答题规范 四大模块。

**放弃原因**：
- 学生进入 3.6.0 时只学过 3.4（求导公式 + 求导法则），**完全没学过"用 f' 判单调"**
- 讲"切点扫描"、"分类讨论"等元方法需要先有判单调的实战经验
- 灌输元方法 = 认知超载，学生学了也记不住

**改为**：从 3.6.1 搬 1.0 + 1.1 过来，让 3.6.0 直接教"怎么判不含参单调性"——**即学即用**。

### 4.2 为什么 3.6.0 不服务 3.5？

3.5 是"点的局部分析"（切线方程），3.6 是"区间的全局分析"（函数性质）——**两条赛道**，方法论不通用。

3.6.0 只服务 3.6.1-3.6.5。

### 4.3 为什么题库和答案要迁移（不复用 3.6.1 的文件）？

用户明确要求：**数据独立**，避免跨页依赖。

- 3.6.0 用 `data/3.6.0/` 下的独立文件
- 3.6.1 的 `data/3.6/` 保留 1.2/1.3/1.4 的题目
- 3.6.0 与 3.6.1 **无 import 交叉**

### 4.4 为什么暂不做"分类讨论决策树"？

用户原始痛点："**分类讨论的时候 不知道咋分类**"。

**延后处理**：分类讨论决策树作为后续优化，建议位置为 **3.6.1 的 1.4 含参讨论开头**。届时学生已熟练判单调性，有实战背景可以承接元方法。

当前 3.6.1 的 1.4 已有 "🗺 4 大题型识别表"（按 `f'` 形态分），暂时够用。
